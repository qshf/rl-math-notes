本章主问题：MC 要等某个 visit 的完整后续回报 $G_t$，DP 又要模型；能不能不知道模型、也不等完整回报算完，每来一步经验就更新价值？
一句话答案：TD 用「一步奖励 + 未来价值估计」构造 target，再用随机近似更新。全章算法共享同一个壳子：旧估计 $+\alpha(\text{target}-\text{旧估计})$，区别只在 target 怎么定义。
问题 1 · TD(0) 给定策略，状态值能不能一步样本就更新？
思路
Bellman equation 说 $v_\pi(s)$ 等于 $R_{t+1}+\gamma v_\pi(S_{t+1})$ 的期望。模型未知时算不了期望，但一步经验给出样本 $r_{t+1},s_{t+1}$，于是用 $r_{t+1}+\gamma v_t(s_{t+1})$ 当 TD target，把 $v_t(s_t)$ 往它那里拉。
结论 $v_{t+1}(s_t)=v_t(s_t)+\alpha_t[\;r_{t+1}+\gamma v_t(s_{t+1})-v_t(s_t)\;]$；每步只更新访问到的 $s_t$
TD 与 MC 的根本分叉：target 从哪里来
方法target何时更新是否自举典型代价
MC完整后续回报 $G_t$episode 结束后反向算 $G_t$，可逐 episode 更新沿途 visit否方差高、不能自然处理 continuing tasks
TD$r_{t+1}+\gamma v_t(s_{t+1})$一步样本到手就更新是，用当前估计接尾巴有 bootstrapping bias

一句话：MC 用真实长回报；TD 用一步真实奖励 + 当前价值估计。

▼新问题：只估 $v(s)$ 还不够做控制，怎么比较动作？
问题 2 · Sarsa 把 TD 从状态值推广到动作值
思路
要改进策略，需要知道每个动作好坏，所以把 $v(s)$ 换成 $q(s,a)$。Sarsa 的 target 使用下一状态下实际采样出来的动作 $a_{t+1}$，样本正好是 $(s_t,a_t,r_{t+1},s_{t+1},a_{t+1})$，因此它评估的是当前正在采样的策略。
结论 Sarsa target：$r_{t+1}+\gamma q_t(s_{t+1},a_{t+1})$；on-policy，可接 $\epsilon$-greedy 做 GPI
▼新问题：一步 TD 太短，MC 又太长，中间能不能调节？
问题 3 · n-step Sarsa 用 $n$ 控制“真实看几步再自举”
思路
位置真实奖励看几步自举尾巴target 长什么样偏差 / 方差
one-step Sarsa1 步立刻用 $q_t(s_{t+1},a_{t+1})$$r_{t+1}+\gamma q_t(s_{t+1},a_{t+1})$偏差较高 / 方差较低
$n$-step Sarsa$n$ 步第 $n$ 步后用 $q_t(s_{t+n},a_{t+n})$$r_{t+1}+\cdots+\gamma^{n-1}r_{t+n}+\gamma^n q_t(s_{t+n},a_{t+n})$中间折中
MC直到 episode 结束不用自举$r_{t+1}+\gamma r_{t+2}+\gamma^2r_{t+3}+\cdots$偏差较低 / 方差较高
结论 $n=1$ 是 Sarsa，$n=\infty$ 是 MC；$n$ 是“真实展开多少步再接估计值”的旋钮。
3-step Sarsa 手算：先看 3 步真实奖励，再接自举尾巴

要更新的是起点这一对 $(s_t,a_t)$。3-step 的意思是：先真的往后看 3 步奖励，再用第 3 步后的动作值估计接上尾巴。

时间位置看到的信息放进 target 的项
$t+1$第 1 步奖励 $r_{t+1}=1$$1$
$t+2$第 2 步奖励 $r_{t+2}=0$$0.9\cdot0$
$t+3$第 3 步奖励 $r_{t+3}=2$$0.9^2\cdot2$
$t+3$ 之后不继续展开真实奖励，用当前表估计 $q_t(s_{t+3},a_{t+3})=5$$0.9^3\cdot5$

通用公式是：

$$ \bar q_t^{(n)} = r_{t+1} +\gamma r_{t+2} +\cdots +\gamma^{n-1}r_{t+n} +\gamma^n q_t(s_{t+n},a_{t+n}) $$

本例取 $n=3$，所以先套成：

$$ \bar q_t^{(3)} = r_{t+1} +\gamma r_{t+2} +\gamma^2 r_{t+3} +\gamma^3 q_t(s_{t+3},a_{t+3}) $$

再代入 $\gamma=0.9,\;\alpha=0.5,\;q_t(s_t,a_t)=2$：

$$ \begin{aligned} \bar q_t^{(3)} &=1+0.9\cdot0+0.9^2\cdot2+0.9^3\cdot5\\ &=1+0+1.62+3.645=6.265,\\ q_{new} &=2+0.5(6.265-2)=4.1325. \end{aligned} $$

读法：$6.265$ 是“看 3 步后的目标值”；$4.1325$ 是把原来的 $q_t(s_t,a_t)=2$ 朝这个目标走一半。实现上不是每隔 $n$ 步才更新一次，而是每来一步，就回头更新“已经等够 $n$ 步信息”的那一对 $(s,a)$。

▼新问题：能不能直接学最优 $q_*$，而不是当前策略的 $q_\pi$？
问题 4 · Q-learning 把下一步实际动作换成下一状态最大动作
思路
Sarsa target 问“当前策略下一步实际选了什么”；Q-learning target 问“如果到下一状态后选当前估计最好的动作，会怎样”。因此它不需要 $a_{t+1}$，只需要 $s_{t+1}$ 后对所有候选动作 $a'$ 取最大。
结论 Q-learning target：$r_{t+1}+\gamma\max_{a'}q_t(s_{t+1},a')$；求 Bellman optimality equation，天然 off-policy
Q-learning 的“off-policy 本质”和“on/off-policy 实现”要分两层看
层次书里的说法行为策略 $\pi_b$：负责采样目标策略 / 被维护的策略TD target
算法本质Q-learning is off-policy可以来自别的策略，只要样本覆盖足够由最优动作值推出 greedy optimal policy$r+\gamma\max_{a'}q(s',a')$
Algorithm 7.2on-policy version / on-policy implementation$\epsilon$-greedy：当前策略 $\pi_t$ 自己采样$\pi_{t+1}$ 仍更新成当前 $q$ 的 $\epsilon$-greedy 策略，不是纯 greedy：
$\pi_{t+1}(a|s_t)= \begin{cases} 1-\epsilon+\epsilon/|\mathcal A(s_t)|,& a=\arg\max_b q_{t+1}(s_t,b)\\ \epsilon/|\mathcal A(s_t)|,& \text{otherwise} \end{cases}$		$r+\gamma\max_{a'}q(s',a')$
Algorithm 7.3off-policy version / off-policy implementation任意探索性 $\pi_b$，可先收集整批经验单独维护 greedy 目标策略 $\pi_T$$r+\gamma\max_{a'}q(s',a')$

所以你截图里的那个可以叫书中 Algorithm 7.2 的 on-policy implementation of Q-learning：on-policy 指“用当前被维护的 $\epsilon$-greedy 策略来采样”，不是说它的 TD target 变成了 Sarsa。若行为策略是 $\epsilon$-greedy、目标策略明确是 greedy，那就是 Algorithm 7.3 这种 off-policy implementation。

同一条经验，Sarsa 和 Q-learning 的 target 可以完全不同

采到 $s_t=A,\;a_t=\text{Right},\;r_{t+1}=1,\;s_{t+1}=B$。在 $B$ 里当前 $q(B,\text{Left})=2,\;q(B,\text{Right})=8$，行为策略实际采样到 $a_{t+1}=\text{Left}$，$\gamma=0.9$：

算法用哪个下一动作target 计算读法
Sarsa实际动作 $a_{t+1}=\text{Left}$$1+0.9\cdot2=2.8$尊重当前策略实际走法
Expected Sarsa按 $\pi(a'|B)$ 加权平均若 $\pi(L)=0.25,\pi(R)=0.75$，则 $1+0.9(0.25\cdot2+0.75\cdot8)=6.85$把下一动作随机性平均掉
Q-learning候选动作最大值 $\max\{2,8\}$$1+0.9\cdot8=8.2$按贪心目标策略看未来
▼新问题：这么多算法，能不能只看一个统一公式？
问题 5 · 统一视角 所有动作值算法只是在换 $\bar q_t$
思路
Sarsa、Expected Sarsa、$n$-step Sarsa、Q-learning、MC 都可以写成同一个随机近似更新：把刚访问的 $q_t(s_t,a_t)$ 朝某个 target $\bar q_t$ 移动。真正区分算法的不是更新壳子，而是 $\bar q_t$ 的定义。
结论 $q_{t+1}(s_t,a_t)=q_t(s_t,a_t)+\alpha_t[\bar q_t-q_t(s_t,a_t)]$

关键公式（结论附带）

本章所有公式都在围绕 target - 当前估计
TD state value一步状态值 prediction
$$v_{t+1}(s_t)=v_t(s_t)+\alpha_t[r_{t+1}+\gamma v_t(s_{t+1})-v_t(s_t)]$$
Sarsa实际下一动作
$$q_{t+1}(s_t,a_t)=q_t(s_t,a_t)+\alpha_t[r_{t+1}+\gamma q_t(s_{t+1},a_{t+1})-q_t(s_t,a_t)]$$
Q-learning下一状态最大动作
$$q_{t+1}(s_t,a_t)=q_t(s_t,a_t)+\alpha_t[r_{t+1}+\gamma\max_{a'}q_t(s_{t+1},a')-q_t(s_t,a_t)]$$
核心洞见
第 7 章把第 5 章的 model-free 采样和第 6 章的随机近似合在一起：用样本构造 Bellman target，然后在线更新。Sarsa 系列主要求给定策略的 Bellman equation，Q-learning 直接求 Bellman optimality equation；它们的灵魂差别都藏在 target 里。
